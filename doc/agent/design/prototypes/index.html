Karmada-Manager 设计原型

基于 Karmada 多集群管理的 Web Dashboard 界面设计原型，提供了两种不同的设计风格： 现代化的 Kuboard 风格和功能完整的原始版本。

🎨 Kuboard 风格版本

参考 Kuboard 优秀的 UI 设计理念，采用现代化的卡片设计、清晰的视觉层次和优雅的交互体验

系统概览 - Kuboard 风格 v2.0

现代化的系统概览界面，采用 Element UI 色彩体系，提供清晰的信息架构和直观的数据展示。 包含系统状态横幅、核心指标卡片、资源使用趋势图表和集群健康状态监控。

Element UI 色彩 现代卡片设计 渐变图标 Chart.js 图表 响应式布局
http://localhost:3000/kuboard-style-overview.html

命名空间管理 - Kuboard 风格 v2.0

参考 Kuboard 命名空间列表的设计，提供现代化的资源管理界面。 包含高级搜索过滤、状态标签、操作按钮和清晰的表格布局。

现代表格设计 智能搜索 状态徽章 分页组件 悬停效果
http://localhost:3000/kuboard-style-namespaces.html

🌟 Kuboard 风格设计特色

• 统一色彩体系：采用 Element UI 的色彩规范，提供一致的视觉体验
• 现代卡片设计：使用圆角、阴影和悬停效果，增强界面层次感
• 清晰信息架构：合理的间距和布局，提高信息的可读性
• 优雅交互体验：流畅的动画过渡和直观的操作反馈

⚙️ 原始功能版本

功能完整的 Karmada 管理界面，包含系统概览、集群管理、调度可视化、策略管理等核心功能模块

系统概览 v1.0

Karmada 控制面状态总览，展示版本信息、运行时间、集群统计、资源使用情况和最近活动。 提供实时的系统健康状态监控和关键指标展示。

实时监控 资源统计 趋势图表 活动时间线
http://localhost:3000/overview.html

集群管理

多集群管理界面，显示所有成员集群的状态、资源使用情况和统计信息。 支持集群搜索、筛选和详细信息查看。

集群卡片 状态监控 资源统计 搜索筛选
http://localhost:3000/clusters.html

工作负载调度可视化

可视化展示工作负载从 Karmada 控制面到成员集群的调度过程。 包含策略选择、副本分配和实时状态展示，帮助用户理解调度决策。

调度流程图 副本分布 实时动画 策略展示
http://localhost:3000/workload-scheduling.html

策略管理

管理 PropagationPolicy、OverridePolicy 和 ClusterPropagationPolicy 等策略。 提供策略创建、编辑、删除和 YAML 查看功能。

策略列表 YAML 编辑 状态监控 搜索筛选
http://localhost:3000/policy-management.html

Pod 调度追溯

详细追溯单个 Pod 的完整调度历程，从 Karmada 控制面的策略匹配到最终在目标节点的运行状态。 包含时间线、相关策略、事件日志和 Pod 详细信息。

调度时间线 策略关联 事件日志 YAML 查看
http://localhost:3000/pod-trace.html

📋 技术实现规范

前端技术栈

  • • HTML5 - 语义化标记结构
  • • Tailwind CSS 3.x - 原子化样式框架
  • • FontAwesome 6.4.0 - 图标库
  • • Chart.js - 数据可视化图表
  • • JavaScript ES6+ - 现代 JavaScript 特性

设计特性

  • • 响应式设计 - 适配多种设备
  • • 现代化UI - 卡片设计和悬停效果
  • • 一致性 - 统一的色彩和间距规范
  • • 可访问性 - 语义化标签和键盘导航
  • • 性能优化 - CDN 资源和代码优化
