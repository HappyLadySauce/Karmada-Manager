# 前端设计修改规范方案文档

## 概述

本文档详细记录了Karmada Dashboard前端的集群资源视图功能实现和相关技术规范。

## 集群资源视图功能实现

### 功能描述
在概览页面底部新增集群资源视图模块，实时展示多云环境中所有集群的状态和资源使用情况。

### 技术实现要点

1. **API接口设计**: `/api/v1/scheduling/clusters/resources`
2. **组件架构**: 响应式集群卡片布局  
3. **数据刷新**: 30秒自动刷新 + 手动刷新
4. **错误处理**: API失败时自动降级到模拟数据
5. **兼容性**: 修复Antd组件deprecated警告

### 核心代码结构

- 新增ClusterCard组件用于展示单个集群信息
- 集成React Query实现数据获取和缓存
- 响应式网格布局适配不同屏幕尺寸
- 状态指示系统（健康/警告/异常）

### API需求文档

已创建完整的API需求文档 `doc/agent/frontend/Demand/API_Demand.md`，包含：

1. **紧急需求接口**:

2. **接口规范**:

3. **开发时间线**:
